# masterji-youtube-style-video-listing

**Objective:**

- Build a YouTube-style video listing page that fetches and displays videos from the FreeAPI YouTube endpoint

**Requirements:**

- Fetch and display a list of YouTube videos
- Show video thumbnails, titles, and channel names
- Click on a video to open it in YouTube
- Add a search bar to filter videos (filteration should be done on frontend without any additional api calls)
- Implement a grid layout for better UI

## Introduction

- The YouTube-style video listing is a web application
- This application fetch youtube videos and display them at homepage to users like youtube homepage

## Scope

- The project aims to provide a basic YouTube Style Video Listing with essential features
- This application will have a simple and user friendly interface to interact between users and application

## Features

- **Accessible** on both mobile devices and computers
- **Responsive** on both mobile devices and computers
- **Grid Layout** on both mobile devices and computers to display videos
- Functionality to **fetch data from API**
- Functionality to **list multiple videos**
- Functionality to **select among multiple videos**
- Functionality to **search among videos** by pressing **Enter Key** or **Search Button**
- Functionality to **list videos as an element**
- Each element have functionality to **open the video on youtube by clicking on video thumbnail**
- Each element have functionality to **open the channel by clicking on channel name**
- Each YouTube Video element that is listed must contain the following information:
  - **Thumbnail** of the video
  - **Channel Name** of the video
  - **Views** of the video
  - **Published Date** of the video

## Important Information

### API

- Application uses `freeapi.app` [**youtube videos fetch api**](https://freeapi.hashnode.space/api-guide/apireference/getYoutubeVideos) to fetch youtube videos and its details
- `freeapi.app` is owned by [**Hitesh Choudhary**](https://github.com/hiteshchoudhary/apihub)

### Responsive

- Application uses certain breakpoints to provide responsiveness among multiple devices that are standerized globally
- These standerized breakpoints are avaliable at [**Tailwind CSS**](https://tailwindcss.com/docs/responsive-design)
